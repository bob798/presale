# 演示
**演示——控场而不是表演**

演示不是讲PPT，是一场信息收集和预期管理的过程。

你讲的每一页，都在观察客户的反应——哪里他认真在听，哪里他分心了，哪里他皱眉，哪里他开始和旁边的人低声说话。这些信号告诉你方案的哪个部分击中了他，哪个部分还有顾虑。

好的演示，是你在讲完之前就已经知道他会提什么问题，已经在方案里埋好了答案。

# 处理焦虑
**处理顾虑——不是说服，是消除**

客户的顾虑分三种，处理方式完全不同。

第一种是信息不足——他不清楚某个细节，给他看清楚就解决了。

第二种是利益顾虑——他担心这个决定对他个人或部门有风险，这种顾虑用逻辑说服不了，要用案例、背书、或者降低他的决策风险来解决。

第三种是借口——他已经决定不买了，顾虑只是他给你的台阶。这种情况继续解答顾虑是浪费时间，要往上找真正的决策人或者识别真实的阻力在哪里。

区分这三种顾虑，比任何话术都重要。
